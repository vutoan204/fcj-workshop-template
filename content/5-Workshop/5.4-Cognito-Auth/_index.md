# Configure Amazon Cognito Authentication

In this section, you will create and configure an Amazon Cognito User Pool to secure user sign-ups and sign-ins, and restrict API Gateway access to authenticated users.

Depending on the region and console state, AWS might present either the **Standard 6-Step Wizard** or the **New Simplified Quick Setup Wizard**. Follow the steps below corresponding to your console view.

---

### Step 1: Create Cognito User Pool

#### Option A: Using the New Simplified Quick Setup Wizard (Recommended)
If you see the **"Set up resources for your application"** page, configure it as follows:
1. **Define your application:**
   * **Application type:** Select **Single-page application (SPA)**. (This is highly important: selecting SPA automatically configures the app client as a *Public Client* with *No Client Secret*, which is required for our client-side React code).
   * **Name your application:** Enter `SmartImage-WebClient`.
2. **Configure options:**
   * **Options for sign-in identifiers:** Check **Email** (ensure username and phone number are unchecked).
   * **Self-registration:** Ensure **Enable self-registration** is checked.
   * **Required attributes for sign-up:** Click the dropdown and select both **email** and **name**.
3. **Add a return URL - optional:**
   * Leave blank (we will use React's custom frontend forms for authentication instead of the Cognito Hosted UI).
4. Click **Create user directory** (this will automatically provision your User Pool and SPA Client).

#### Option B: Using the Standard 6-Step Wizard
If you see the multi-step setup page:
1. Under **Configure sign-up**:
   * **Cognito user pool sign-in options:** Select **Email** only. Click **Next**.
2. Under **Configure security requirements**:
   * **Password policy:** Choose **Cognito defaults**.
   * **Multi-factor authentication (MFA):** Select **No MFA**. Click **Next**.
3. Under **Configure sign-up experience**:
   * Keep **Self-service sign-up** checked.
   * **Required attributes:** Choose **name** and **email**. Click **Next**.
4. Under **Configure message delivery**:
   * **Email provider:** Select **Send email with Cognito**. Click **Next**.
5. Under **Integrate app**:
   * **User pool name:** Enter `SmartImage-UserPool`.
   * **Hosted UI:** Unchecked.
   * **App client:** Select **Public client**.
   * **App client name:** Enter `SmartImage-WebClient`.
   * **Client secret:** Select **Don't generate a client secret** (crucial for SPA React integration).
   * Click **Next**.
6. Under **Review and create**: Click **Create user pool**.

### Step 2: Add Custom Attributes

We need to add a custom attribute `custom:role` to track whether a user is a regular user or an admin.
1. In the Cognito Console, select your newly created user pool (e.g., `SmartImage-UserPool` or the pool generated by the quick setup).
2. Navigate to the **Sign-in experience** tab.
3. Scroll down to **Custom attributes** and click **Add custom attribute** (or **Create custom attribute**).
4. **Attribute name:** Enter `role`.
5. **Type:** Select **String**.
6. **Constraints:** Set Min length to `1` and Max length to `20`.
7. Click **Save changes** (or **Create**).

---

### Step 3: Create User Groups

We will define two administrative groups:
1. Open the user pool in the console and navigate to the **User groups** tab.
2. Click **Create group**.
3. **Group name:** Enter `admin`.
4. **Description:** Enter `Administrators with moderation access`.
5. **Precedence:** Enter `0` (highest priority).
6. Click **Create group**.
7. Click **Create group** again.
8. **Group name:** Enter `user`.
9. **Description:** Enter `Regular users`.
10. **Precedence:** Enter `10`.
11. Click **Create group**.

![Cognito Groups Setup](/images/5-Workshop/5.4-Cognito-Auth/cognito_groups_setup.png)

*Note: Save your **User Pool ID** and **App Client ID** from the User Pool overview page. You will need them when configuring the API Gateway authorizer and React frontend.*
