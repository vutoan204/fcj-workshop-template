# Workshop Overview: Smart Image Platform

The **Smart Image Platform** is an AWS Serverless & Event-Driven application designed to automate image uploading, processing (resizing, thumbnail generation), and AI-based image classification.

In this workshop, you will learn how to build and configure the entire infrastructure of this platform **step-by-step using the AWS Management Console**.

### Solution Architecture

The platform orchestrates Serverless cloud services to streamline chronological data flows, as illustrated in the system execution layout below:

![Smart Image Platform Architecture](/images/2-Proposal/platform_architecture.jpeg)

* **Key Services Used:**
  * **Amazon S3:** Object storage to store raw (ingestion) and processed (resized/thumbnails) images.
  * **Amazon Cognito:** User directory and authentication layer (User Pool & App Client).
  * **Amazon DynamoDB:** NoSQL database storing image metadata (using single-table design with GSIs and Stream enabled), user profile information, and upload quotas.
  * **AWS Lambda:**
    * `api-handler`: Handles REST API requests (presigned URL generation, user profile CRUD, quota verification).
    * `image-processor`: Triggered by S3 raw uploads (prefix `users/`) to generate thumbnails and save metadata.
    * `ai-analyzer`: Triggered by DynamoDB stream to extract labels via Amazon Rekognition.
  * **Amazon API Gateway:** Exposes backend services via REST APIs.
  * **Amazon Rekognition:** Automatically analyzes images to detect objects, scenes, and content safety.
  * **AWS Amplify:** Hosts the React frontend application.
  * **S3 Presigned GET URLs:** Secure, time-bound URLs (generated by Lambda via API Gateway) used to load and download assets, temporarily replacing public CDN distribution (CloudFront) due to account restriction safeguards.

### Step-by-Step Workshop Steps
In the following sections, you will build this architecture using the AWS Console:
1. **Prerequisites:** Get access keys, repo, and environment ready.
2. **Storage & Database:** Create raw and processed S3 buckets, and configure the DynamoDB tables (`SmartImage-Images` with GSI and Stream, `SmartImage-UserQuotas`, and `SmartImage-UserProfiles`).
3. **Cognito Authentication:** Setup User Pools to secure client requests.
4. **Backend Serverless:** Write Lambda functions, configure execution roles, hook up S3 triggers (prefix `users/`) and DynamoDB Streams, and expose via API Gateway.
5. **Frontend Deploy:** Deploy the React web application with AWS Amplify.
6. **Verification & Testing:** Verify the end-to-end flow from upload to AI classification.
